Intro

This anonymous message board is my take on the final project of the freeCodeCamp courses.

Feel free to explore the app and its API, adding/deleting/reporting threads and replies to as many boards as you please.

Above and Beyond

As this is the last project in freeCodeCamp's curriculum , I've chosen to enjoy myself and take the project a few steps beyond the basic user story requirements.

Here are the key additional features I've worked on:

  • I've designed the project to be more mobile-friendly, making this my first design for mobile first project. Ideally, I'd like to make the aesthetic of the project visually "lighter", but I'm already well beyond the project requirements and, for now at least, it's time to ship this puppy.

    The app/site is fully responsive and will resize itself to match your screen size. Feel free to visit this project on your mobile device, where you should see that the mock phone bezel (visible on a computer) has disappeared.

    Alternatively, if you're on a computer, you can use your browser's developer tools to switch to "device view" (in Firefox and Chrome, press F12 on your keyboard, then click on the little icon that looks like a phone and a tablet together in the top row of the developer panel). Depending on your browser, you might have to refresh the page to see the design correctly, but you should see the phone bezel disappear when you switch to a device-mimicking view. The bezel itself is pure CSS.
  • I've written server-side form data validation for the key form fields. Even if the fields in all the forms are set to "required", a sneaky user who turns off this setting in the developer tools should not be able to break the API by submitting faulty data or empty fields.
  • I've striven to incorporate web accessibility for this project from the start, adding titles and labels everywhere in an effort to be inclusive and provide all users equal access to the information and functionality of the site/app.
  • I've chosen to make the alert messages prescribed by the user stories more informative and thorough in order to improve the user experience.
  • I've added a home page that dynamically lists all the existing message boards and provides links to each one.
  • That silly little clock in the status bar tells time. It was a fun little bit of code to bang out over a cup of tea at breakfast. ^_^
  • I've written Chai assertion tests (with iteration loops, I have 134 test, all passing) in order to thoroughly test the project's functionality.
  • I've also checked the spelling on the different pages.

Code & Tech Stack

What you see here was designed and coded by me (Julian NF). If you have a look at my code and see my "we" comments, don't be alarmed; it's how I like to comment in order to make my code more approachable and teachable.

I chose not to use any external scripts or code snippets (e.g. for menus, sliders, etc.), and in fact the technology stack that I used is limited to the following:

  • JavaScript
  • jQuery and AJAX
  • Node.js
  • Express.js
  • MongoDB
  • Mocha.js and Chai.js for testing
  • Helmet.js
  • a little sprinkling :) of CSS.
What remains of the original project-specific boilerplate code for this project is comprised mostly of code that freeCodeCamp implements to test finished projects.

Those bits of the original boilerplate that remain and are not for testing, I've tidied up to simplify them and to ensure that this code is as close to best-practice as possible. For example:

  • correcting single quotes where standards say there should be double quotes, such as in HTML tag attributes
  • refactoring code, expanding functionality, and combining similar code, such as with the form submission functions
  • ensuring that all elements on a page have unique ID attributes, particularly for dynamically added content
  • updating package versions for the project and removing any unused and unnecessary ones.
I haven't written or updated the Mocha-launching test-runner.js file which came with the boilerplate.
